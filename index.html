Understanding

고객의 니즈와 문제점을 정확히 파악하는 단계입니다.

Co-creation Workshop

고객과 함께 창의적인 워크샵을 통해 문제를 정의하고 해결 방안을 모색합니다.

주요 활동

  • 고객 인터뷰 및 관찰
  • 문제 정의 및 우선순위 설정
  • 해결 방안 브레인스토밍
  • 프로토타입 제작 및 테스트

Desk Research

기존 데이터와 연구 결과를 분석하여 시장 동향과 경쟁사 정보를 수집합니다.

주요 활동

  • 시장 조사 및 분석
  • 경쟁사 벤치마킹
  • 트렌드 분석
  • 데이터 기반 인사이트 도출

Discover

사용자의 니즈와 행동을 깊이 있게 이해하고 인사이트를 도출하는 단계입니다.

Focus Group Interview

소수의 참여자와 함께 그룹 토론을 통해 사용자의 의견과 경험을 수집합니다.

주요 활동

  • 그룹 토론 진행
  • 참여자 의견 수집
  • 토론 내용 분석
  • 인사이트 도출

In-depth Interview

개별 사용자와의 심층 인터뷰를 통해 상세한 피드백과 경험을 수집합니다.

주요 활동

  • 1:1 심층 인터뷰
  • 사용자 경험 탐색
  • 상세 피드백 수집
  • 인터뷰 내용 분석

Contextual Inquiry

사용자의 실제 환경에서 관찰과 인터뷰를 통해 자연스러운 사용 패턴을 파악합니다.

주요 활동

  • 현장 관찰
  • 맥락 기반 인터뷰
  • 사용 패턴 분석
  • 개선점 도출

Service Safari

서비스 이용 과정을 직접 체험하며 사용자 경험을 분석합니다.

주요 활동

  • 서비스 체험
  • 경험 기록
  • 문제점 파악
  • 개선 방안 도출

Expert Interview

도메인 전문가와의 인터뷰를 통해 전문적인 인사이트를 수집합니다.

주요 활동

  • 전문가 인터뷰
  • 전문 지식 수집
  • 인사이트 도출
  • 전략 수립

Usability Test

제품이나 서비스의 사용성을 테스트하여 개선점을 발견합니다.

주요 활동

  • 사용성 테스트
  • 사용자 행동 관찰
  • 문제점 기록
  • 개선 방안 도출

Persona

목표 사용자의 특성과 니즈를 구체화하여 페르소나를 작성합니다.

주요 활동

  • 사용자 특성 분석
  • 페르소나 작성
  • 니즈 정의
  • 목표 설정

Journey map

사용자의 서비스 이용 과정을 시각화하여 경험을 분석합니다.

주요 활동

  • 여정 단계 정의
  • 감정 변화 기록
  • 터치포인트 분석
  • 개선 기회 발견

Log Analytics

사용자의 서비스 이용 로그를 분석하여 행동 패턴을 파악합니다.

주요 활동

  • 로그 데이터 수집
  • 행동 패턴 분석
  • 이용 통계 도출
  • 개선점 발견

Voice of Customer

고객의 피드백과 의견을 수집하여 서비스 개선에 활용합니다.

주요 활동

  • 고객 피드백 수집
  • 의견 분석
  • 개선점 도출
  • 우선순위 설정

Eyetracking

사용자의 시선 추적을 통해 인터페이스의 효과성을 분석합니다.

주요 활동

  • 시선 추적 실험
  • 시선 패턴 분석
  • 인터페이스 평가
  • 개선 방안 도출

Survey

대규모 사용자 대상 설문을 통해 정량적 데이터를 수집합니다.

주요 활동

  • 설문지 작성
  • 데이터 수집
  • 통계 분석
  • 결과 도출

Voting

사용자들의 투표를 통해 의견을 수집하고 우선순위를 결정합니다.

주요 활동

  • 투표 항목 설정
  • 투표 진행
  • 결과 분석
  • 우선순위 결정

Define

수집된 데이터를 분석하고 해결해야 할 문제를 명확히 정의하는 단계입니다.

Affinity Diagram

다양한 아이디어와 데이터를 그룹화하여 패턴과 관계를 파악합니다.

주요 활동

  • 데이터 수집 및 정리
  • 그룹화 및 분류
  • 패턴 분석
  • 인사이트 도출

Card Sorting

사용자의 인지 모델을 이해하기 위해 정보를 분류하고 구조화합니다.

주요 활동

  • 카드 작성
  • 분류 작업
  • 구조 분석
  • 정보 구조 설계

Mind Mapping

아이디어와 개념을 시각적으로 연결하여 전체적인 구조를 파악합니다.

주요 활동

  • 중심 개념 정의
  • 관련 개념 연결
  • 구조 시각화
  • 관계성 분석

Heuristics

사용성 평가를 위한 경험적 방법론을 적용하여 문제점을 발견합니다.

주요 활동

  • 평가 기준 설정
  • 문제점 식별
  • 심각도 평가
  • 개선 방안 도출

Positioning Map

제품이나 서비스의 시장 위치를 시각화하여 경쟁 우위를 파악합니다.

주요 활동

  • 포지셔닝 요소 정의
  • 경쟁사 분석
  • 맵 작성
  • 전략 수립

Develop

정의된 문제를 해결하기 위한 솔루션을 개발하고 구현하는 단계입니다.

Mental Model

사용자의 인지 모델을 기반으로 서비스 구조를 설계합니다.

주요 활동

  • 사용자 인지 분석
  • 모델 구축
  • 서비스 구조 설계
  • 구현 방안 도출

Expert Solutions

전문가의 지식과 경험을 활용하여 최적의 솔루션을 도출합니다.

주요 활동

  • 전문가 의견 수집
  • 솔루션 기획
  • 타당성 검증
  • 구현 계획 수립

Deliver & Validation

개발된 솔루션을 검증하고 최종 결과물을 전달하는 단계입니다.

Service Prototyping

서비스의 핵심 가치와 경험을 빠르게 구현하여 검증합니다.

주요 활동

  • 프로토타입 제작
  • 사용자 테스트
  • 피드백 수집
  • 개선 및 반영

UX Flow

사용자 경험의 흐름을 설계하고 최적화합니다.

주요 활동

  • 사용자 여정 설계
  • 인터랙션 정의
  • 흐름 최적화
  • 구현 가이드 작성

Flow Chart

서비스의 프로세스와 로직을 시각화하여 명확히 합니다.

주요 활동

  • 프로세스 분석
  • 플로우 작성
  • 로직 검증
  • 문서화

A/B Testing

다양한 버전을 비교하여 최적의 솔루션을 도출합니다.

주요 활동

  • 테스트 설계
  • 데이터 수집
  • 결과 분석
  • 최적화
